# 基于Transformer与图卷积网络的多罪名预测研究

作者：学生姓名

摘要：
本文提出并实现了一种基于Transformer编码器与图卷积网络（GCN）相结合的多罪名（multi-label）预测模型，用以解决司法文本中存在的多标签、类别不均衡和标签间相关性问题。模型采用Transformer提取案件事实文本的深层语义特征，同时利用罪名之间的共现关系构造标签图并通过GCN进行标签信息传播，最终通过多标签分类器输出每个罪名的预测概率。我们在内部数据集上训练并测试模型，训练10轮后在验证集上达到micro-F1=0.979844、macro-F1=0.136891、accuracy=0.963769，最终损失为0.001096（见训练日志）。实验结果表明：模型在常见罪名上表现优异（高micro-F1与accuracy），但在长尾稀有罪名上仍存在较大挑战（低macro-F1），显示出数据不均衡对宏观指标的影响。

关键词：Transformer；图卷积网络；多罪名预测；多标签分类；司法文本

1. 引言

随着自然语言处理（NLP）技术在司法领域的应用不断深入，自动判定案件相关罪名的研究受到广泛关注。司法文本通常具有语言复杂、事实陈述冗长且罪名可能同时适用于同一案情的特点，因此多标签分类成为更贴合实际需求的任务。此外，罪名之间往往存在语义或共现关系（例如某些罪名常常同时出现），这为利用标签之间结构化信息提升预测性能提供了机会。

本文提出的模型结合了Transformer强大的文本表示能力与GCN对标签图结构信息的建模能力：Transformer负责从案件事实文本中抽取高质量的语义向量表示，GCN则在构造的罪名图上进行标签间信息传播，从而在多标签预测时考虑标签相关性。我们在项目数据集上进行了训练与评估，并报告最终训练日志中的关键指标以便复现与比对。

2. 相关工作

近年来，基于深度学习的司法文本分析工作逐渐增多。Transformer（如BERT、RoBERTa等）在文本表示方面表现优异，被广泛用于法律文本的分类与抽取任务。另一方面，图神经网络（GNN）尤其是图卷积网络（GCN）被证明在建模标签关系、知识图推理等场景中效果显著。将文本编码器与标签图网络相结合的框架已在多标签文本分类中得到应用，本研究在司法多罪名预测的背景下采用该思路并结合工程化训练流程进行验证。

3. 方法

3.1 模型概述

模型由三部分组成：文本编码器、标签图构建与GCN、以及多标签分类器。

- 文本编码器：使用Transformer编码器对案件事实文本进行编码，得到句/文档级别的语义向量。
- 标签图与GCN：基于训练语料统计罪名共现关系构建标签邻接矩阵，并将标签表示初始化后通过GCN进行若干层信息传播，从而获得融合了邻域信息的标签表示。
- 多标签分类器：将文本表示与每个标签的表示进行交互（例如拼接或点积），并通过sigmoid输出每个罪名的预测概率，采用二元交叉熵损失（binary cross-entropy）进行训练。

3.2 损失函数与训练策略

采用逐标签的二元交叉熵作为训练目标，整体损失为各标签损失的加权和（本实现中权重均一）。考虑到类别不均衡问题，可选地对长尾类别采用重采样或加权损失，但本版本保留均一权重以记录基线表现。

4. 实验

4.1 数据集

数据集来源于项目内部构建的司法案件文本，包含案件事实描述及对应的多罪名标签。数据集中罪名分布存在明显不均衡：若干常见罪名样本丰富，而多数罕见罪名样本稀少。

4.2 实验设置

模型训练进行了10轮（epoch=10）。训练过程的关键终点日志记录在训练目录下的日志文件中（路径：checkpoints/logs/20260326152945/epoch_10_metrics.txt）。我们在验证集上评估精度（accuracy）、micro-F1与macro-F1等指标以衡量多标签分类性能。

4.3 评估指标

使用如下常用多标签评估指标：

- Accuracy（样本级别准确率）：若模型对样本预测的标签集合与真实标签集合完全一致，则计为正确。
- micro-F1：在所有标签的预测结果上计算全局的精确率与召回率，再综合成F1分数，强调对所有实例和标签的总体性能，易受常见标签影响。
- macro-F1：对每个标签分别计算F1，再对标签取均值，能够反映各标签（尤其稀有标签）的平均性能，不受类别大小影响。

5. 结果与分析

训练日志（epoch 10）记录的关键结果如下：

- epoch = 10
- loss = 0.001096
- micro_f1 = 0.979844
- macro_f1 = 0.136891
- accuracy = 0.963769

上述结果表明：

- 模型在整体样本级别和常见标签上的表现非常好，micro-F1与accuracy都接近0.96–0.98的高值，说明模型能够较准确地识别出多数常见罪名。
- 但macro-F1仅为0.136891，明显偏低，说明在标签维度上存在明显的不均衡：多数罕见罪名的F1很低，拉低了宏平均性能。这一现象通常由训练数据的标签长尾分布导致，且当前训练中未对长尾类别进行额外增强或加权处理。

综合来看，模型在捕捉主流罪名方面效果显著，表明Transformer编码对司法文本语义的提取能力良好，且GCN对标签共现的建模在一定程度上提升了多标签预测的性能；但要在所有罪名上达到均衡性能，还需进一步针对数据不均衡和少样本标签采取策略，例如：

- 对长尾罪名进行过采样或使用数据增强技术；
- 在损失函数中加入类别权重或采用Focal Loss等对难例加权的方法；
- 采用标签平衡的微调策略或基于外部知识增强标签表示；
- 采用更复杂的标签依赖建模（例如条件随机场或自回归标签生成）以更好地捕获标签间的结构化关系。

6. 结论与未来工作

本文实现并验证了一种基于Transformer与GCN相结合的多罪名预测框架，在内部数据集上取得了高micro-F1与高accuracy，但macro-F1较低，反映出数据不均衡对少样本罪名预测的挑战。未来工作将主要集中在缓解类别不均衡与提升长尾罪名性能的策略上，包括数据增强、重加权损失、半监督学习或外部知识整合等方向。
